# find command searches editable project sources

`mops find` searches **Editable Project Sources** by default because its intended caller is an LLM looking for model code it may modify, not packaged project-visible libraries or stub models. The command has two explicit modes: `find instances` returns matching root or child nodes for any resolvable fully qualified concept name, including subconcepts by default and excluding them with `--exact`; `find usages` returns references to any resolvable target node. Both modes default to source locations from **Editable Project Sources**, use bounded single-response results with `--limit`, and keep response payloads domain-shaped as reusable `nodes` and `usages` responses rather than one generic search-result format.

`--all` widens either mode to the whole **MPS Repository** (**Project and Libraries**), so a caller that needs to see where a concept or node is used across read-only library and stub models — not just the sources it can edit — can opt into that broader scope explicitly. The default stays narrow so the common editing case is not drowned in library matches.
