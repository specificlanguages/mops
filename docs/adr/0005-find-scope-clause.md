# find scopes searches with a trailing `in` clause, and explicit scopes are exhaustive

`mops find` subcommands narrow their **Search Scope** with a trailing keyword clause — `find instances <concept> in <scope-segments>` — rather than an option like `--in <value>`. The clause reuses the `list` navigation-path grammar verbatim (ADR 0002): everything after the first freestanding `in` argument is scope segments, so multi-segment paths, serialized references, and segment-level ambiguity reporting all carry over without a second grammar. An option value could not hold multiple segments without inventing an escaping scheme, and a parallel single-string grammar would inevitably drift from the path grammar.

Scope semantics follow one rule: the default scope is **Editable Project Sources** (ADR 0003), and an explicit scope is searched exhaustively — including read-only libraries and stubs within it — because naming a scope is already an intentional act. This made the `--all` flag redundant; it was removed in favor of `in /`, keeping exactly one way to say "whole repository". Anyone tempted to reintroduce `--all` or a `--in` option should read this first: both were considered and rejected to keep one navigation grammar and one scope rule.
